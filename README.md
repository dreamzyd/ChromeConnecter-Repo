# ChromeConnecter

远程浏览器控制 Chrome 扩展 + 中继服务器。这套系统能让部署在云端的 AI Agent（如 OpenClaw）通过 WebSocket 安全地操控你本地的 Chrome 浏览器，完美解决云服务器 IP 被各大网站（如微信公众号、知乎等）反爬虫系统拦截的痛点。

## 架构说明

```text
用户本地 PC                         2号服务器                 1号服务器（云端）
┌──────────────────┐    WSS     ┌──────────────┐   内网WS   ┌─────────────────────┐
│  Chrome Extension │ ────────→ │  OpenResty    │ ────────→ │  Relay WS  :18793   │
│  service-worker   │ ←──────── │  :443 反向代理 │ ←──────── │                     │
│  chrome.debugger  │           └──────────────┘           │  Relay HTTP :18794  │
└──────────────────┘                                       │   (仅 localhost)     │
                                                            │                     │
                                                            │  OpenClaw  :18789   │
                                                            └─────────────────────┘
```

- **Chrome Extension**：运行在用户本地浏览器中，通过 `chrome.debugger` API 执行页面操控指令，模拟真实的家庭宽带与浏览器指纹。
- **Relay Server**：部署在云服务器，作为中继桥梁。对外开放 WebSocket 端口（18793）接收扩展连接；对内开放 HTTP 端口（18794，仅限 localhost）供 AI Agent 调用。
- **安全机制**：采用 Google Authenticator TOTP 动态口令验证，确保只有授权的用户才能控制本地浏览器。

## 1. 下载与安装部署

### 1.1 部署云端中继服务器 (Relay Server)
你需要一台安装了 Node.js 环境的云服务器。

1. 克隆本项目到云服务器：
   ```bash
   git clone <你的 GitHub 仓库地址>
   cd ChromeConnecter/relay-server
   ```
2. 安装依赖并启动服务：
   ```bash
   npm install
   npm start
   ```
   > **注意：** 服务启动后，请务必在云服务器的安全组/防火墙中放行 **TCP 18793** 端口。

### 1.2 安装本地 Chrome 扩展
1. 将本项目中的 `extension` 文件夹下载到你的本地电脑。
2. 打开 Chrome 浏览器，在地址栏输入 `chrome://extensions/` 进入扩展管理页面。
3. 开启页面右上角的 **“开发者模式”** (Developer mode)。
4. 点击左上角的 **“加载已解压的扩展程序”** (Load unpacked)，选择刚才下载的 `extension` 文件夹。

### 1.3 下载身份验证器应用 (Authenticator)
由于系统强制使用 TOTP 动态验证码来保障安全，你需要在手机上下载一个兼容的应用：
- **方案 A**：手机自带应用商店搜索并下载 `Microsoft Authenticator`（国内安卓极力推荐，无网络限制）。
- **方案 B**：下载 `Google Authenticator`。
- **方案 C**：使用阿里云 App 内置的“虚拟 MFA”功能。

## 2. 连接与验证指南

1. **配置服务器地址**：点击本地 Chrome 浏览器右上角的 `ChromeConnecter` 扩展图标，在最底部的“中继服务器地址”框中填入：
   ```text
   ws://<你的云服务器公网IP>:18793
   ```
2. **等待连接成功**：填入正确的地址且服务器端口已放行后，扩展顶部的状态会从黄色的“连接中”变成绿色的 **“已连接”**。
3. **获取绑定二维码**：连接成功后，扩展面板上会自动刷新出一个二维码。
4. **手机扫码绑定**：
   - 打开手机上下载好的 Authenticator 应用。
   - 点击“+”号或“添加账号”，选择“扫描二维码”。
   - 扫描扩展面板上的二维码。
   *(如果二维码无法加载，也可在云端通过 API 获取密钥手动添加，详见下文)*
5. **完成验证**：手机应用上会出现 6 位动态验证码（每30秒刷新）。将这 6 位数字填入扩展面板，点击 **“验证”**。
6. 验证通过后，扩展会提示“会话有效”（默认保持 8 小时）。在这 8 小时内，云端的 AI Agent 就可以自由下发指令了。

---

## 3. AI Agent (OpenClaw) 接口调用
当本地浏览器处于验证通过状态时，云主机的 AI Agent 就可以通过调用 `http://127.0.0.1:18794` 来下发浏览器操作指令了。

详细的 API 接口文档和能力清单，请查阅本项目中的 `skill/SKILL.md` 文件。

### 常见问题与排错 (FAQ)
- **Q: 扩展一直显示黄色的“连接中”？**
  A: 检查服务器 IP 是否填错，以及云主机提供商（如阿里云/腾讯云）控制台的“安全组”规则中是否放行了 18793 端口。
- **Q: 二维码刷不出来，如何手动绑定 TOTP？**
  A: 在云主机终端运行 `curl http://127.0.0.1:18794/api/totp/setup`，会返回一个包含 `secret` 字段的 JSON。在手机 Authenticator 应用中选择“手动输入密钥”，填入该 secret 即可生成动态密码。
